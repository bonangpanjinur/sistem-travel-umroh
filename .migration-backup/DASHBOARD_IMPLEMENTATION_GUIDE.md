# Panduan Implementasi Role-Based Dashboard\n\n## Ringkasan\n\nDokumen ini menjelaskan implementasi sistem dashboard berbasis peran (Role-Based Dashboard) untuk aplikasi VinsTourTravel. Sistem ini memungkinkan setiap peran pengguna memiliki dashboard yang disesuaikan dengan tugas dan tanggung jawab mereka.\n\n## Arsitektur Sistem\n\n### Komponen Utama\n\n#### 1. **Base Dashboard Template** (`BaseDashboardTemplate.tsx`)\n- Komponen reusable untuk semua dashboard\n- Menyediakan struktur umum: header, filter, quick actions, stats cards, alerts\n- Responsive dan dapat dikustomisasi\n\n#### 2. **Dashboard Modules**\nLima dashboard spesifik per peran:\n- **BranchManagerDashboard**: Untuk manajer cabang\n- **FinanceDashboard**: Untuk tim keuangan\n- **SalesDashboard**: Untuk tim sales\n- **MarketingDashboard**: Untuk tim marketing\n- **EquipmentDashboard**: Untuk tim perlengkapan\n\n#### 3. **Configuration System** (`dashboard-config.ts`)\n- Registry modul dashboard (DASHBOARD_MODULES)\n- Konfigurasi per peran (ROLE_DASHBOARD_CONFIG)\n- Helper functions untuk validasi akses\n\n#### 4. **Access Management**\n- **useDashboardAccess.ts**: Hook untuk manajemen akses\n- **useDashboardRouter.ts**: Hook untuk routing dinamis\n- **DashboardAccessManager.tsx**: UI untuk super_admin\n- **DashboardProtectedRoute.tsx**: Route guard\n\n#### 5. **Database Schema**\n- `dashboard_access_config`: Menyimpan konfigurasi akses per peran\n- `dashboard_access_audit_log`: Audit trail untuk perubahan konfigurasi\n\n## Alur Implementasi\n\n### Fase 1: Setup Database\n\n1. **Jalankan migration SQL**\n   ```bash\n   # Buka Supabase dashboard\n   # Buka SQL Editor\n   # Copy-paste konten dari src/lib/migrations/dashboard-access-config.sql\n   # Jalankan query\n   ```\n\n2. **Verifikasi tabel dibuat**\n   - `dashboard_access_config`\n   - `dashboard_access_audit_log`\n\n### Fase 2: Integrasi Routes\n\n1. **Update AdminRoutes.tsx** (sudah dilakukan)\n   - Import semua dashboard baru\n   - Tambahkan route untuk setiap dashboard\n   - Tambahkan route untuk DashboardAccessManager\n\n2. **Update menu registry** (opsional)\n   - Tambahkan menu items untuk dashboard baru di `admin-menu-registry.ts`\n   - Update `useDynamicMenus.ts` jika diperlukan\n\n### Fase 3: Integrasi Auth & Access Control\n\n1. **Update ProtectedRoute.tsx** (opsional)\n   - Tambahkan DashboardProtectedRoute untuk dashboard-specific routes\n   - Atau gunakan middleware di route definition\n\n2. **Testing akses per peran**\n   - Login sebagai super_admin → akses /admin/dashboard-access\n   - Login sebagai branch_manager → akses /admin/branch-manager\n   - Login sebagai finance → akses /admin/finance-dashboard\n   - dll.\n\n### Fase 4: Konfigurasi Default\n\n1. **Verifikasi default configuration**\n   - Buka Supabase dashboard\n   - Cek tabel `dashboard_access_config`\n   - Pastikan semua peran memiliki konfigurasi default\n\n2. **Customize jika diperlukan**\n   - Super admin dapat mengakses /admin/dashboard-access\n   - Ubah enabled_modules dan default_dashboard per peran\n\n## File-File Baru\n\n### Components\n```\nsrc/components/dashboards/\n├── BaseDashboardTemplate.tsx       # Base template untuk semua dashboard\n├── DashboardProtectedRoute.tsx      # Route guard untuk dashboard\n```\n\n### Pages\n```\nsrc/pages/admin/\n├── DashboardAccessManager.tsx       # UI untuk super_admin\n├── DashboardRedirect.tsx            # Redirect ke dashboard sesuai peran\n├── dashboards/\n│   ├── BranchManagerDashboard.tsx   # Dashboard untuk branch manager\n│   ├── FinanceDashboard.tsx         # Dashboard untuk finance\n│   ├── SalesDashboard.tsx           # Dashboard untuk sales\n│   ├── MarketingDashboard.tsx       # Dashboard untuk marketing\n│   └── EquipmentDashboard.tsx       # Dashboard untuk equipment\n```\n\n### Hooks\n```\nsrc/hooks/dashboards/\n├── useDashboardAccess.ts            # Hook untuk manajemen akses\n└── useDashboardRouter.ts            # Hook untuk routing dinamis\n```\n\n### Libraries\n```\nsrc/lib/\n├── dashboard-config.ts              # Konfigurasi dashboard\n└── migrations/\n    └── dashboard-access-config.sql  # Database migration\n```\n\n## Penggunaan\n\n### Untuk Super Admin\n\n1. **Akses Dashboard Access Manager**\n   - Navigasi ke `/admin/dashboard-access`\n   - Pilih peran dari dropdown\n   - Enable/disable modul untuk peran tersebut\n   - Set default dashboard\n   - Perubahan disimpan otomatis\n\n2. **Lihat Audit Trail**\n   - Klik tab \"Audit Log\"\n   - Lihat riwayat perubahan konfigurasi\n   - Lihat siapa yang membuat perubahan dan kapan\n\n### Untuk Staff User\n\n1. **Login**\n   - Sistem otomatis mengarahkan ke dashboard default untuk peran mereka\n   - Contoh: branch_manager → /admin/branch-manager\n\n2. **Akses Dashboard**\n   - Hanya dapat mengakses modul yang di-enable untuk peran mereka\n   - Jika mencoba akses modul yang di-disable → Access Denied\n\n## Konfigurasi Default\n\n### Super Admin & Owner\n- Akses semua dashboard\n- Default: Admin Main Dashboard\n\n### Branch Manager\n- Akses: Branch Manager Dashboard\n- Default: Branch Manager Dashboard\n\n### Finance\n- Akses: Finance Dashboard\n- Default: Finance Dashboard\n\n### Sales\n- Akses: Sales Dashboard\n- Default: Sales Dashboard\n\n### Marketing\n- Akses: Marketing Dashboard\n- Default: Marketing Dashboard\n\n### Equipment\n- Akses: Equipment Dashboard\n- Default: Equipment Dashboard\n\n### Operational\n- Akses: Operational Dashboard\n- Default: Operational Dashboard\n\n### Agent\n- Akses: Agent Dashboard\n- Default: Agent Dashboard\n\n### Customer\n- Akses: Customer Dashboard\n- Default: Customer Dashboard\n\n## Customization\n\n### Menambah Dashboard Baru\n\n1. **Buat komponen dashboard**\n   ```tsx\n   // src/pages/admin/dashboards/NewDashboard.tsx\n   import BaseDashboardTemplate from '@/components/dashboards/BaseDashboardTemplate';\n   \n   export default function NewDashboard() {\n     return (\n       <BaseDashboardTemplate\n         title=\"Dashboard Baru\"\n         subtitle=\"Deskripsi dashboard\"\n         statsCards={[...]}\n         quickActions={[...]}\n         alerts={[...]}\n       >\n         {/* Custom content */}\n       </BaseDashboardTemplate>\n     );\n   }\n   ```\n\n2. **Tambah ke dashboard-config.ts**\n   ```ts\n   export const DASHBOARD_MODULES: Record<string, DashboardModule> = {\n     new_dashboard: {\n       id: 'new-dashboard',\n       key: 'new_dashboard',\n       label: 'Dashboard Baru',\n       description: 'Deskripsi',\n       component: 'NewDashboard',\n       icon: 'Icon',\n       requiredPermission: 'new_dashboard',\n       requiredRoles: ['new_role'],\n     },\n   };\n   ```\n\n3. **Tambah route di AdminRoutes.tsx**\n   ```tsx\n   const NewDashboard = lazy(() => import('@/pages/admin/dashboards/NewDashboard'));\n   \n   // Di dalam route definition\n   <Route path=\"new-dashboard\" element={<LazyPage><NewDashboard /></LazyPage>} />\n   ```\n\n4. **Update database**\n   ```sql\n   INSERT INTO dashboard_access_config (role, enabled_modules, default_dashboard)\n   VALUES ('new_role', ARRAY['new_dashboard'], 'new_dashboard')\n   ON CONFLICT (role) DO UPDATE SET enabled_modules = array_append(excluded.enabled_modules, 'new_dashboard');\n   ```\n\n## Best Practices\n\n1. **Gunakan BaseDashboardTemplate**\n   - Konsistensi desain di semua dashboard\n   - Mudah untuk maintenance\n\n2. **Optimize Data Fetching**\n   - Gunakan React Query untuk caching\n   - Set staleTime dan gcTime yang sesuai\n   - Hindari fetch data yang tidak perlu\n\n3. **Audit Trail**\n   - Semua perubahan konfigurasi tercatat\n   - Super admin dapat melihat riwayat perubahan\n\n4. **Error Handling**\n   - Tampilkan pesan error yang jelas\n   - Gunakan DashboardProtectedRoute untuk access control\n\n5. **Performance**\n   - Lazy load dashboard components\n   - Gunakan Suspense untuk loading states\n   - Memoize expensive computations\n\n## Troubleshooting\n\n### Dashboard tidak muncul\n1. Verifikasi route sudah ditambahkan di AdminRoutes.tsx\n2. Cek console untuk error messages\n3. Pastikan component import path benar\n\n### Access Denied error\n1. Verifikasi user memiliki role yang sesuai\n2. Cek dashboard_access_config di database\n3. Pastikan modul di-enable untuk role tersebut\n\n### Data tidak muncul di dashboard\n1. Verifikasi query di Supabase\n2. Cek RLS policies di database\n3. Pastikan user memiliki permission untuk akses tabel\n\n## Testing\n\n### Manual Testing\n\n1. **Test role-based access**\n   ```\n   - Login sebagai super_admin\n   - Akses /admin/dashboard-access\n   - Verify dapat mengubah konfigurasi\n   \n   - Login sebagai branch_manager\n   - Akses /admin/branch-manager\n   - Verify dashboard muncul dengan data yang benar\n   - Coba akses /admin/finance-dashboard\n   - Verify Access Denied\n   ```\n\n2. **Test audit trail**\n   ```\n   - Login sebagai super_admin\n   - Ubah konfigurasi di /admin/dashboard-access\n   - Lihat audit log\n   - Verify perubahan tercatat dengan benar\n   ```\n\n3. **Test default dashboard redirect**\n   ```\n   - Login sebagai berbagai role\n   - Verify otomatis redirect ke default dashboard\n   - Verify default dashboard sesuai dengan role\n   ```\n\n## Maintenance\n\n### Regular Tasks\n1. Monitor audit log untuk aktivitas mencurigakan\n2. Update dashboard configuration sesuai kebutuhan bisnis\n3. Optimize query performance jika diperlukan\n4. Backup database secara regular\n\n### Updates\n1. Saat menambah role baru, tambahkan entry di dashboard_access_config\n2. Saat menambah dashboard baru, update DASHBOARD_MODULES dan ROLE_DASHBOARD_CONFIG\n3. Test semua perubahan di development environment sebelum production\n\n## Referensi\n\n- [React Query Documentation](https://tanstack.com/query/latest)\n- [Supabase RLS Documentation](https://supabase.com/docs/guides/auth/row-level-security)\n- [React Router Documentation](https://reactrouter.com/)\n- [TypeScript Documentation](https://www.typescriptlang.org/)\n
